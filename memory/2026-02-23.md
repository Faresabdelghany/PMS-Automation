## PMS — Design Fix + Agent Network (Omar subagent)

### Critical lesson learned
Fares was frustrated because Sprint 3 Tasks board didn't match the existing design.
**Rule**: Before writing ANY UI code in PMS, read these files first:
- `docs/design-system.json` — ALL design tokens (source of truth)
- `docs/design_concept.json` — design philosophy
- `components/project-board-view.tsx` — Kanban gold standard
- `components/projects-content.tsx` — page structure gold standard

### What was done
1. **Tasks board design fixed** (`components/tasks/TasksBoard.tsx`, `app/(dashboard)/tasks/page.tsx`)
   - Added `bg-background mx-2 my-2 border border-border rounded-lg` envelope to tasks page (matches Projects)
   - Columns: `rounded-xl bg-muted` (was `bg-accent/20 border border-border/40`)
   - Column icons: Phosphor icons (was colored dots)
   - "New Task" button: `variant="ghost" size="sm"` (was hardcoded purple)
   - All borders: `border-border` not `border-border/40`

2. **Agent Network page built** (`app/(dashboard)/agents/communication/page.tsx`)
   - Server + Client component combo

3. **Sidebar updated** — "Agent Network" under Agents → `/agents/communication`

4. **Build result**: 0 TypeScript errors, 35/35 pages, `/agents/communication` + `/tasks` both in route table

### Key design tokens to remember
- Page envelope: `flex flex-1 flex-col bg-background mx-2 my-2 border border-border rounded-lg min-w-0`
- Kanban column: `rounded-xl bg-muted`
- Column header buttons: `h-7 w-7 rounded-lg` ghost
- Filter chips: `bg-accent/50 text-muted-foreground` inactive, `bg-accent text-foreground` active
- NO hardcoded colors ever — only CSS custom properties
