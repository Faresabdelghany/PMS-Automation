# Platform Launch Marketing Plan (PMS)

**Task:** Platform launch marketing plan — channels, messaging, timeline, assets  
**Constraints:** No outbound posting without explicit approval. Keep concise & execution-ready.  
**ICP:** Tech Lead / Head of Eng at 5–30 person software agency.  
**Positioning:** The first PM tool built for teams running AI agents.

---

## 1) Launch Objective
Generate qualified signups and demo interest from agency engineering leaders by clearly framing PMS as the only PM tool that treats AI agents as first-class teammates.

**Primary KPI:** Qualified signups / demo requests from ICP  
**Secondary KPIs:** Launch page conversion rate, email CTR, activation to first project created.

---

## 2) Messaging Pillars (with copy)
**Pillar A — AI agents are first‑class teammates**  
- **Hero Headline:** *“The first PM tool built for teams running AI agents.”*  
- **Subhead:** *“Plan, assign, and track work across humans and agents—without hacks or plugins.”*  
- **CTA:** *“Start a Project”* / *“Book a 15‑min Demo”*

**Pillar B — Built for agency delivery**  
- **Value Line:** *“Ship client work faster with clear ownership, status, and execution—across small teams.”*  
- **CTA:** *“See how agencies run PMS”*

**Pillar C — Replace tool sprawl**  
- **Value Line:** *“Stop stitching Jira + agents + docs. Run everything in one system.”*  
- **CTA:** *“See the workflow”*

> **Note:** Keep capability claims high‑level (agents as first‑class citizens). Avoid feature specifics until confirmed.

---

## 3) Channel Plan (no outbound without approval)
**Owned**
1. **Launch page** (website) — single landing page focused on ICP and positioning.
2. **Email to waitlist / existing leads** — 1 announcement + 1 follow‑up.
3. **In‑product banner** (if any logged‑in users) — “New platform launch” CTA to landing page.

**Earned / Partner (prepare drafts, do not publish)**
1. **Founder LinkedIn post** (draft only).
2. **Relevant communities** (draft only; e.g., agency leadership, AI tooling groups).

**Paid (optional, post‑approval)**
- **Small retargeting set** to launch page visitors / site traffic.

---

## 4) Timeline (Execution‑Ready)
**T‑14 to T‑7 (Prep)**
- Finalize launch page copy + creative.
- Assemble product screenshots + simple demo GIF/video.
- Draft outbound assets (email + social). **Do not send/publish.**

**T‑6 to T‑2 (QA & alignment)**
- Review copy claims vs confirmed capabilities.
- Align on CTA (self‑serve vs demo).
- Stakeholder approval gate for any public release.

**T‑1 (Pre‑launch)**
- Set analytics events: landing page view, CTA click, signup submit.
- QA page on mobile/desktop.

**Launch Day (T‑0)**
- Publish launch page.
- Send approved email.
- Optional: publish approved social/community posts.

**T+1 to T+14 (Follow‑up)**
- Follow‑up email: “How agencies run agents in PMS” + case‑style story.
- Monitor conversion performance; iterate headline/CTA if needed.
